# Agent 与 ReAct 框架

## 什么是 Agent

Agent（智能体）是指能够感知环境、自主决策并调用工具完成复杂任务的大模型应用形态。与传统的单轮问答不同，Agent 具备多步推理、工具调用和状态管理能力。

## ReAct 框架

ReAct（Reasoning + Acting）是一种经典的 Agent 范式，通过交替进行推理（Thought）和行动（Action）来解决问题：

1. **Thought（思考）**：大模型分析当前状态，决定下一步该做什么
2. **Action（行动）**：调用外部工具（如搜索、计算、数据库查询）
3. **Observation（观察）**：获取工具返回结果，进入下一轮思考

这一循环持续进行，直到问题被解决或达到最大步数。

## 工具调用

Agent 的核心能力是工具调用（Tool Use / Function Calling）。常见工具包括：
- 知识检索工具（RAG）
- 网络搜索工具
- 代码执行工具
- 数据库查询工具

大模型根据用户意图自动选择合适的工具，并组织调用顺序。

## 与 RAG 的关系

Agent 和 RAG 常结合使用：Agent 编排业务流程，RAG 作为其中的知识检索工具被按需调用，实现更复杂的智能应用。
