# Prompt Engineering 提示工程

## 什么是提示工程

Prompt Engineering（提示工程）是通过设计、优化输入给大模型的提示词，引导模型生成更准确、更符合预期输出的技术。它是低成本提升大模型效果的重要手段。

## 常用技巧

### 1. 角色设定
为模型设定明确角色，如"你是一位资深 Python 工程师"，约束输出风格和专业性。

### 2. 少样本学习（Few-Shot Learning）
在提示中提供几个示例，让模型学习输入输出的模式。分为：
- Zero-shot：不提供示例
- One-shot：提供一个示例
- Few-shot：提供多个示例

### 3. 思维链（Chain of Thought, CoT）
要求模型逐步展示推理过程，如"让我们一步一步思考"，显著提升数学推理和复杂逻辑任务的表现。

### 4. 指令明确化
使用清晰、具体的指令，避免歧义。如"请用 3 点总结，每点不超过 50 字"。

## 提示词优化原则

- 明确性：指令清晰无歧义
- 结构化：使用分隔符、列表组织内容
- 示例引导：用 Few-shot 展示期望输出
- 迭代测试：反复测试和调整

## 与微调的区别

提示工程不改模型参数，成本低、迭代快；微调修改模型权重，效果持久但成本高。两者常结合使用。
